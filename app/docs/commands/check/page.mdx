# cxgrd check

Validates your project structurally and with compiler-backed checks. Catches circular dependencies, orphaned files, layer violations, and type/syntax errors.

## Usage

```bash
cxgrd check .
cxgrd check . --staged        # only staged files (great for pre-commit hooks)
cxgrd check . --strict        # fail if compiler tools are missing (recommended for CI)
```

> Compiler backed verification works only if the compilers for the particular programming language are installed on the system.

## Options

`--staged` : Only checks git staged files

`--changed` : Only checks staged and unstaged changed files

`--skip-compiler` : Skip compiler-backed verification

`--skip-structural` : Skip structural graph checks

`--strict` : Conduct strict checks on all files. Fail if a detected language compiler was skipped

`--ci` : CI mode: post result to server for GitHub PR status (Team plan). 

The `--ci` option requires some efforts. It is a GitHub Actions workflow step, so you need to configure CI pipeline with a yaml file:

```bash
name: cxgrd check

on:
  pull_request:
    branches: [main]

jobs:
  architecture:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install cxgrd
        run: npm install -g cxgrd

      - name: Scan project
        run: cxgrd scan

      - name: cxgrd architecture check
        run: cxgrd check --ci --strict
        env:
          CXGRD_AUTH_TOKEN: ${{ secrets.CXGRD_TOKEN }}
```

CXGRD_AUTH_TOKEN is the team member's cxgrd JWT stored as a GitHub Actions secret. The dev adds it once under repo settings → Secrets → Actions.

So when a PR is opened,
```
GitHub Actions picks it up
  → installs cxgrd
    → scans the repo (builds .cg/ graph)
      → runs check --ci --strict
        → posts result to /api/teams/:id/ci-check
          → server updates GitHub commit status (✓ or ✗ on the PR)
```
